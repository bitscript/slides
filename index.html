Static Authentication

Static Authentication

  • many options
  • collect user's info (password, phone number, secret questions, email address, etc.)
  • login: issues challenge(s) / follow complex protocols
  • verifies user response, cross fingers

The Blockchain gives us an alternative.

Scriptable Authentication

  • User defines identity using Bitcoin Script
  • Collect user's blockchain identity
  • User provides unlocking script
  • Service executes the scripts
This is something that was nearly impossible to coordinate around the world, until a consensus could be made around a authentication scripting language.

Stateless Script Evaluation


import { execute, OpCode } from 'bitcoin-script'

const a = execute(
  { blockHeight: 0, blockTime: 0, nLockTime: 0,
  nSequence: 0, stack: [], altStack: [], programCounter: 0,
  serializedScript: new Uint8Array([
  OpCode.OP_2,
  OpCode.OP_3,
  OpCode.OP_ADD
  ])})

a.next(); a.next(); a.next();
